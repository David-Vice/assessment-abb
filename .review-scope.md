# Review Scope — ABB RAG, Phase P1 (Foundations)

This project is **not yet under git**, so there is no base-branch diff. Treat the
**entire current source tree as new code** (the P1 foundation scaffold). Review it
as you would a first PR that introduces the project skeleton.

## What this phase intentionally is
P1 is the **foundation only**: monorepo wiring, shared contracts, service skeletons
(health endpoints), DB schema, Docker/compose, CI, conventions. RAG logic
(chunking, embeddings, retrieval, rerank, generation), the scraper internals, the
real frontend UI, the dashboard, and the eval harness are **deliberately deferred**
to later phases (P2–P8). Do NOT flag "missing feature X" if X belongs to a later
phase per `.plans/`. DO flag anything that will make those phases harder, any
foundation defect, inconsistency, type/safety issue, or convention violation.

## Review mandate
Judge against: correctness, type safety, security, consistency with `CONVENTIONS.md`,
Docker/compose correctness, pgvector/SQL correctness (note: `halfvec(3072)` chosen
because HNSW caps `vector` at 2000 dims), uv-workspace correctness, and whether the
foundation sets up P2–P8 cleanly. Flag real issues with file:line and a concrete fix.
